# CLAUDE.md - 计划执行约定

## 1. 范围控制原则

### 1.1 严格按指令执行
- 用户说"只改方案" → 只修改文档/方案，不写代码
- 用户说"写代码" → 才写代码
- **绝不擅自扩展范围**

### 1.2 禁止过度执行
- 不提供未被请求的示例代码
- 不创建未被请求的文件
- 不修改未被请求的内容

---

## 2. 修改完整性检查

### 2.1 修改前
- 明确修改范围和内容
- 简要说明打算做什么

### 2.2 修改后（必须）
- 通读修改后的全文
- 检查前后逻辑一致性
- 确保没有矛盾或冲突的描述
- 简要说明改了什么

### 2.3 一致性检查清单
- [ ] 新增的概念是否在全文中统一
- [ ] 旧描述是否需要同步更新
- [ ] 是否存在相互矛盾的表述

## 3. 执行保证

1. **严格按范围执行** - 不多做也不少做
2. **修改后全面检查** - 确保逻辑自洽
3. **发现问题立即修正** - 不遗留问题到下一步
